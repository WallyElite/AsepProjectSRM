1: theta = sin(theta) allows for small angle aproximation. However, at theta = 0, even though at equilibrium, the system is still unstable. Why is this? The reason is all due to its own momentum and the fact that there is no real way to reach theta = 0 exactly as p, i, and d would need to be super specific. And in the case of this day 1 assignment, the parameters are set at numbers which wouldn't allow perfect pid values in the first place. Positive feedback plays a role in this as if p was any bigger, than there will be more error, as the pendulum will swing past theta = 0. This causes an oscillation that dampens each time where d tries to dampen kp but also ki tries to track accumulated error and reapply it, where eventually the pendulum should be roughly theta = 0.

3: When the pendulum starts to tilt away from theta = 0, the PID controller will notice an error in angle and depending on what Kp is set to, the pendulum will fix that error more aggresively and faster. Ki is the surefire way to remove accumulated error by increasing output on how much corerction there should be to Kd is there to dampen the output of error correction by kp. Many cases, without kd, kp will cause overshooting if too fast. But without high kp, it might take too long for correction to happen. So having a high kp and a relative high kd will cause fast and accurate control.

4: Random accumulated error naturally builds up from noise towards the gyro. The gyro sensor is terrible at moment to moment tracking although great on an average. So to prevent continuous build up of error, 98% of the actual previous gyro data is established as position and the other 2% is from any new build up which is usually very small so thats why its 2% and not 50%.
